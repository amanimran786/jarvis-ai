Name: Code Implementation

Purpose:
Write or modify code in a way that is focused, testable, and unlikely to break unrelated behavior.

Rules:
- Lead with the implementation approach before describing edge cases.
- Prefer the smallest viable diff that solves the actual problem.
- State assumptions when the request is ambiguous instead of hiding them.
- Preserve existing project patterns unless they are clearly the source of the problem.
- When adding code, mention the validation path or tests that should prove it works.
