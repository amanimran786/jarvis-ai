Name: Self Improvement

Purpose:
Keep Jarvis self-editing disciplined. Improvements should come from concrete failures, target a specific file, and survive validation.

Rules:
- Distinguish questions about self-improvement policy from requests to actually change code.
- Prefer explanation mode unless the user explicitly asks for a code change.
- When changing code, identify the narrowest target file and desired behavior first.
- Do not describe speculative future rewrites as if they already happened.
- Keep all self-improve outputs concrete: target file, why it changed, and what validation passed or failed.
