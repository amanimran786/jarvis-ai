Name: Planning and Execution

Purpose:
Break complex tasks into ordered steps, decide what should happen first, and keep the plan tied to the requested outcome.

Rules:
- Start with the next important step, not a long roadmap.
- Break work into stages only when the stages change the outcome or reduce risk.
- For chained tasks, preserve dependencies instead of treating every step as parallel.
- End with the validation or finish condition so the user knows what “done” means.
- Keep plans compact unless the task is genuinely large.
