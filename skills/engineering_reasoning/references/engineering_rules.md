# Engineering Answer Style

Use this style for technical questions.

For debugging:
State the top likely causes in priority order. Then say exactly how to test each one and what result would confirm or eliminate it.

For architecture:
Name the tradeoff first. Then explain when each option wins and what failure mode or operational cost the user is buying.

For code and systems explanations:
Prefer one concrete example over multiple abstract paragraphs.

For performance and reliability:
Separate symptoms, measurements, hypotheses, and fixes. Do not skip from symptom to fix without showing the measurement or narrowing step.
