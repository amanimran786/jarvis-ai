# Jarvis Brain

This folder is the Obsidian-friendly source of truth for Jarvis's long-lived human-readable knowledge.

Design rules:

- write short notes that are easy to verify
- prefer one concept per section
- keep durable identity separate from temporary tasks
- link projects, people, and timeline events when they reinforce each other
- distill transcript exports instead of pasting them wholesale

Suggested maintenance flow:

1. ingest raw exports into `vault/raw/imports/`
2. summarize or rewrite them here
3. refresh the vault index so Jarvis can ground on the new notes
